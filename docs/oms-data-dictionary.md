# Sprint Connect OMS — Data Dictionary

**Version:** 2.0  
**Database:** MySQL (per-module schemas)

Plain-language definitions for every table and column in the OMS data model.

---

## Module 1 — Order (schema: `orders`)

---

### `orders` — the order record

One row per customer order. This is the central aggregate that all other tables in this module relate to.

| Column | Type | Plain meaning |
|---|---|---|
| `order_id` | UUID PK | OMS's internal unique ID for the order. |
| `order_number` | varchar UK | The human-readable reference shown to staff and customers — e.g. `ORD-001`. Generated by OMS. |
| `source_order_id` | varchar | The order reference from the originating system (e.g. the app, POS, or e-commerce platform). Used to deduplicate order creation — if this ID already exists, OMS returns the existing order. |
| `channel_type` | varchar | Where the order originated: `Web`, `App`, `POS`, `CallCenter`. |
| `business_unit` | varchar | The brand operating this order — e.g. `TOPS`, `CFH`. |
| `store_id` | UUID FK | The store or DC fulfilling this order. References `config.store_locations`. |
| `fulfillment_type` | varchar | How the order will reach the customer: `Delivery` (driver delivers), `Express` (faster delivery), `ClickAndCollect` (customer picks up at store). |
| `payment_method` | varchar | How the customer pays: `Prepaid` (paid before delivery), `PostPaid` (paid after), `CreditCard`. |
| `status` | varchar | The current stage of the order in the lifecycle state machine. |
| `pre_hold_status` | varchar | The status the order was in before it was put on hold. Saved so it can be restored when the hold is released. `null` when not on hold. |
| `hold_reason` | varchar | Why the order was placed on hold — e.g. `ManualReview`, `PackageDamaged`. `null` when not on hold. |
| `substitution_flag` | bool | `true` if WMS has proposed at least one substitution on this order. Never reset to `false`. |
| `pos_recalc_pending` | bool | `true` if the order is waiting for POS to confirm the final price after a quantity change or substitution. The order cannot proceed to `Packed` while this is `true`. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |
| `created_by` / `updated_by` | varchar | User or system that created or last changed the record. |

---

### `order_lines` — individual products on the order

One row per product requested. An order with 5 different SKUs has 5 rows here.

| Column | Type | Plain meaning |
|---|---|---|
| `order_line_id` | UUID PK | Unique ID for this line. |
| `order_id` | UUID FK | The order this line belongs to. |
| `sku` | varchar | Product code from the catalogue. |
| `product_name` | nvarchar | Product display name, copied from the catalogue at order creation so the name is preserved even if the catalogue changes later. |
| `barcode` | varchar | Barcode for scanning at the warehouse during picking. |
| `unit_of_measure` | varchar | How the product is counted or weighed: `Each`, `Kg`, `Litre`. |
| `requested_amount` | decimal | How many units the customer ordered. |
| `picked_amount` | decimal | How many units were actually picked by the warehouse. Set when WMS confirms pick. May be less than `requested_amount` if stock ran out. |
| `original_unit_price` | decimal | The price per unit at the time the order was placed. |
| `status` | varchar | `Active` (normal) or `Voided` (line removed because customer rejected a substitution). |

---

### `order_line_substitutions` — proposed product replacements

One row per substitution offered by the warehouse picker. A substitution is offered when the picker cannot find the original SKU and proposes an alternative.

| Column | Type | Plain meaning |
|---|---|---|
| `substitution_id` | UUID PK | Unique ID for this substitution. |
| `order_line_id` | UUID FK | The original order line being substituted. |
| `substitute_sku` | varchar | The replacement product the picker is proposing. |
| `substitute_product_name` | nvarchar | Display name of the replacement. |
| `substitute_unit_price` | decimal | Price of the replacement product. |
| `substituted_amount` | decimal | Quantity of the replacement being offered. |
| `customer_approved` | bool \| null | The customer's decision: `true` (accepted), `false` (rejected), `null` (not yet decided). |
| `approved_at` | timestamptz | When the customer made their decision. Set on either approval or rejection. |
| `created_at` | timestamptz | When the picker offered the substitution. |

---

### `order_packages` — physical packages the order is shipped in

One row per package. A large order may be split across multiple packages.

| Column | Type | Plain meaning |
|---|---|---|
| `package_id` | UUID PK | Unique ID for this package. |
| `order_id` | UUID FK | The order this package belongs to. |
| `tracking_id` | varchar | The TMS tracking code assigned to this package — e.g. `TRK-2024-001`. This is the key used to match incoming TMS webhooks. |
| `vehicle_type` | varchar | What vehicle will carry this package — `Van`, `Motorbike`, `Truck`. |
| `package_weight` | decimal | Weight in kilograms. Used for carrier billing and routing. |
| `status` | varchar | Package delivery status: `OutForDelivery`, `Delivered`, `Damaged`. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |

---

### `order_package_lines` — which order lines are in which package

A join table linking each order line to its package. Allows one order to be split across multiple packages.

| Column | Type | Plain meaning |
|---|---|---|
| `package_line_id` | UUID PK | Unique ID for this assignment. |
| `package_id` | UUID FK | The package. |
| `order_line_id` | UUID FK | The order line packed inside it. |

---

### `order_addresses` — customer and delivery addresses

One row per address on the order. Currently used for the delivery address; can hold billing or pickup address if needed (distinguished by `address_type`).

| Column | Type | Plain meaning |
|---|---|---|
| `address_id` | UUID PK | Unique ID. |
| `order_id` | UUID FK | The order this address belongs to. |
| `address_type` | varchar | `Delivery`, `Billing`, or `Pickup`. |
| `first_name` / `last_name` | varchar | Recipient name. |
| `mobile_phone` | varchar | Contact number for the delivery driver. |
| `email` | varchar | Contact email. |
| `address1` | nvarchar | Street address. |
| `subdistrict` / `district` / `province` / `postal_code` | varchar | Thai address hierarchy. |

---

### `order_customers` — customer profile snapshot

A snapshot of the customer's profile at the time of order. Preserved even if the customer later changes their account details.

| Column | Type | Plain meaning |
|---|---|---|
| `customer_id` | UUID PK | Unique ID for this snapshot. |
| `order_id` | UUID FK | The order this snapshot is for. |
| `external_customer_id` | varchar | Customer ID from the loyalty or membership system. |
| `name` | varchar | Customer display name. |
| `email` | varchar | Customer email at time of order. |
| `phone` | varchar | Customer phone at time of order. |

---

### `delivery_slots` — the scheduled delivery window

One row per order. Stores when the delivery is expected to happen.

| Column | Type | Plain meaning |
|---|---|---|
| `slot_id` | UUID PK | Unique ID. |
| `order_id` | UUID FK | The order this slot is for. |
| `store_id` | UUID FK | The store or DC dispatching the delivery. |
| `scheduled_start` | timestamptz | Start of the delivery window — e.g. `14:00`. |
| `scheduled_end` | timestamptz | End of the delivery window — e.g. `16:00`. |
| `booked_via` | varchar | Which system booked the slot: `TMS` (Prepaid booking), `WMS`, `Manual`. |
| `booking_ref` | varchar | The reference number from the booking system. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |

---

### `order_holds` — hold audit log

An append-only log of every hold and release action on an order. Useful for auditing why an order was delayed.

| Column | Type | Plain meaning |
|---|---|---|
| `hold_id` | UUID PK | Unique ID for this hold event. |
| `order_id` | UUID FK | The order that was held. |
| `hold_reason` | varchar | Why the order was put on hold. |
| `held_at` | timestamptz | When the hold was placed. |
| `held_by` | varchar | Who placed the hold — a user ID or system name. |
| `released_at` | timestamptz | When the hold was released. `null` if still on hold. |
| `released_by` | varchar | Who released the hold. `null` if still on hold. |

---

### `order_outbox` — staging table for outbound domain events

Every domain event OMS needs to send to an external system (WMS, TMS, POS, GW) is written here first, atomically with the domain state change. A background worker reads pending rows and dispatches them.

| Column | Type | Plain meaning |
|---|---|---|
| `outbox_id` | UUID PK | Unique ID. |
| `order_id` | UUID FK | The order this event relates to. |
| `event_type` | varchar | What happened — e.g. `OrderCreatedEvent`, `PickConfirmedEvent`, `OrderCancelledEvent`. |
| `event_payload` | jsonb | The full event data to be dispatched. |
| `status` | varchar | `Pending` (not yet sent), `Published` (sent successfully), `Failed` (dispatch failed after retries). |
| `created_at` | timestamptz | When the event was staged. |
| `published_at` | timestamptz | When the background worker successfully dispatched it. `null` if still pending. |

---

### `order_status_history` — immutable status transition log

An append-only record of every time the order's status changed. Used to build the order timeline and for audit purposes.

| Column | Type | Plain meaning |
|---|---|---|
| `history_id` | UUID PK | Unique ID for this log entry. |
| `order_id` | UUID FK | The order that changed status. |
| `from_status` | varchar | The status before the transition. `null` for the first entry (Pending). |
| `to_status` | varchar | The new status after the transition. |
| `changed_at` | timestamptz | Exact time of the transition. |
| `changed_by` | varchar | Who or what caused the change — a user ID, or a system name like `WMS`. |
| `detail` | nvarchar | Human-readable note explaining what happened at this step. |

---

### `order_webhook_logs` — log of every inbound webhook received

Every webhook callback received from WMS, TMS, or POS is logged here, even if it results in no state change. Used for debugging, the order timeline, and the idempotency check.

| Column | Type | Plain meaning |
|---|---|---|
| `webhook_log_id` | UUID PK | Unique ID for this log entry. |
| `order_id` | UUID FK | The order this webhook relates to. |
| `source_system` | varchar | Which external system sent this callback: `WMS`, `TMS`, `POS`. |
| `event_type` | varchar | The specific event name — e.g. `PickStarted`, `PackageDelivered`. |
| `detail` | nvarchar | Human-readable summary of what the webhook contained. |
| `raw_payload` | jsonb | The full JSON body of the webhook request, preserved for debugging. |
| `idempotency_key` | varchar UK | The `X-Idempotency-Key` from the request header. If a duplicate key is received, the second request is ignored. |
| `received_at` | timestamptz | When OMS received the webhook. |

## Module 2 — Payment (schema: `payment`)

---

### `order_payments` — payment record per order

The top-level payment record linking an order to its payment status.

| Column | Type | Plain meaning |
|---|---|---|
| `payment_id` | UUID PK | Unique payment ID. |
| `order_id` | uuid | The order this payment covers. |
| `payment_method` | varchar | How the customer pays: `CreditCard`, `BankTransfer`, `StoreCredit`. |
| `total_amount` | decimal | The total amount to be collected. |
| `currency` | varchar | Currency code — `THB`. |
| `status` | varchar | `Pending`, `Authorised`, `Captured`, `Refunded`, `Failed`. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |

---

### `payment_transactions` — individual payment gateway calls

One row per payment gateway transaction. An order may have multiple transactions — an initial authorisation, a capture, and possibly a refund.

| Column | Type | Plain meaning |
|---|---|---|
| `transaction_id` | UUID PK | Unique ID for this transaction. |
| `payment_id` | UUID FK | The payment this transaction belongs to. |
| `amount` | decimal | Amount of this specific transaction. |
| `currency` | varchar | Currency. |
| `payment_method` | varchar | The payment instrument used for this transaction. |
| `gateway_ref` | varchar | The transaction reference returned by the payment gateway. |
| `created_at` | timestamptz | When the transaction was submitted. |

---

### `invoices` — fiscal invoices issued to customers

One invoice per order. Created when the order is delivered or collected.

| Column | Type | Plain meaning |
|---|---|---|
| `invoice_id` | UUID PK | OMS's unique invoice ID. |
| `order_id` | uuid | The order this invoice is for. |
| `invoice_number` | varchar UK | The fiscal invoice number — e.g. `INV-2024-001`. |
| `invoice_type` | varchar | `Standard` (regular invoice), `ABB` (pre-delivery Prepaid invoice sent to WMS), `TaxInvoice` (official tax document from STS). |
| `total_amount` | decimal | Total amount invoiced. |
| `currency` | varchar | `THB`. |
| `status` | varchar | `Generated` (created in OMS), `Issued` (sent to customer), `Voided` (cancelled by a credit note). |
| `generated_at` | timestamptz | When OMS created the invoice record. |
| `issued_at` | timestamptz | When the invoice was sent to the customer. |

---

### `credit_notes` — invoices partially or fully reversed

Issued when a return reduces the amount the customer owes. One credit note per return.

| Column | Type | Plain meaning |
|---|---|---|
| `credit_note_id` | UUID PK | Unique ID. |
| `invoice_id` | UUID FK | The invoice being partially reversed. |
| `credit_note_number` | varchar UK | The fiscal credit note number — e.g. `CN-RET-001`. |
| `amount` | decimal | Amount being credited back to the customer. |
| `currency` | varchar | `THB`. |
| `reason` | varchar | Why the credit note was issued: `Return`, `Discount`. |
| `status` | varchar | `Issued` (created), `Applied` (applied to the payment). |
| `issued_at` | timestamptz | When the credit note was issued. |

---

### `order_line_amounts` — recalculated price per line, per round

One row per recalculation round per order line. When WMS substitutes a product or picks a different quantity, POS recalculates the price. Each round creates a new set of rows here.

| Column | Type | Plain meaning |
|---|---|---|
| `amount_id` | UUID PK | Unique ID. |
| `order_line_id` | UUID FK | The order line this calculation is for. |
| `recalc_round` | int | Which recalculation this is — starts at 1, increments each time POS recalculates. |
| `trigger_event` | varchar | What caused this recalculation — `PickConfirmed`, `SubstitutionApproved`, `ManualTrigger`. |
| `original_unit_price` | decimal | Price before this recalculation — used to show the difference. |
| `recalculated_unit_price` | decimal | Final price after promotions applied by POS. |
| `unit_net_amount` | decimal | Per-unit price after tax deducted. |
| `recalculated_at` | timestamptz | When POS performed this recalculation. |
| `created_at` | timestamptz | When the record was saved in OMS. |

---

### `order_line_taxes` — tax breakdown per amount record

Breaks the tax portion of a line amount into individual tax components (VAT, excise tax, etc.).

| Column | Type | Plain meaning |
|---|---|---|
| `tax_id` | UUID PK | Unique ID. |
| `amount_id` | UUID FK | The price calculation this tax belongs to. References `order_line_amounts`. |
| `tax_type` | varchar | Tax category: `VAT`, `ExciseTax`, `WithholdingTax`. |
| `tax_description` | varchar | Human-readable name — e.g. `Value Added Tax 7%`. |
| `amount` | decimal | The tax amount in currency for this line. |
| `rate` | decimal | The tax rate as a decimal — e.g. `0.07` for 7% VAT. |
| `created_at` | timestamptz | When the record was created. |

---

### `order_fees` — additional charges on an order

Fees added on top of product prices — delivery fees, service charges, platform fees.

| Column | Type | Plain meaning |
|---|---|---|
| `fee_id` | UUID PK | Unique ID. |
| `order_id` | uuid | The order this fee is charged on. |
| `source_fee_id` | varchar | Fee reference from POS or billing system. |
| `fee_code` | varchar | Machine-readable code — e.g. `DELIVERY_FEE`, `SERVICE_CHARGE`. |
| `fee_name` | nvarchar | Human-readable name shown on the receipt — e.g. `Delivery Fee`. |
| `fee_type` | varchar | Category: `Delivery`, `Service`, `Platform`. |
| `amount` | decimal | Fee amount charged. |
| `currency` | varchar | Currency. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |

---

### `order_promotions` — discounts applied to an order

One row per promotion applied. An order can have multiple promotions (e.g. a member discount and a product-specific offer).

| Column | Type | Plain meaning |
|---|---|---|
| `promotion_id` | UUID PK | Unique ID. |
| `order_id` | uuid | The order the promotion was applied to. |
| `order_line_id` | uuid \| null | The specific line the discount reduces. `null` if the promotion applies to the whole order. |
| `source_promo_id` | varchar | Promotion ID from POS or promotions engine. |
| `promo_code` | varchar | The promotion code — e.g. `FRESH10`, `MEMBER20`. |
| `promo_name` | nvarchar | Human-readable name — e.g. `10% Fresh Produce Discount`. |
| `promo_type` | varchar | How the discount works: `PercentageDiscount`, `FixedDiscount`, `BuyXGetY`, `FreeGift`. |
| `discount_amount` | decimal | The actual money deducted by this promotion. |
| `discount_percentage` | decimal | The percentage rate if applicable — e.g. `0.10` for 10%. |
| `currency` | varchar | Currency. |
| `created_at` | timestamptz | When the promotion was applied. |

---

## Module 3 — Returns (schema: `returns`)

---

### `returns` — the return request

One row per return. The top-level record tracking a customer return from request to refund.

| Column | Type | Plain meaning |
|---|---|---|
| `return_id` | UUID PK | OMS's unique ID for this return. |
| `order_id` | uuid | The original order the customer is returning items from. |
| `return_order_number` | varchar UK | Human-readable reference shown to staff and customers — e.g. `RET-001`. |
| `invoice_id` | varchar | The invoice being partially reversed by this return. |
| `credit_note_id` | varchar | The credit note issued to the customer when the return is processed. |
| `status` | varchar | Current stage: `Requested → PickupScheduled → PickedUp → Received → Inspected → PutAway → Refunded`. |
| `goods_receive_no` | varchar | The warehouse receipt number issued when the returned goods physically arrive at the dock. |
| `return_reason` | varchar | Why the customer is returning — e.g. `WrongItem`, `Damaged`, `ChangeOfMind`, `QualityIssue`. |
| `requested_at` | timestamptz | When the customer submitted the return request. |
| `pickup_scheduled_at` | timestamptz | When a driver is scheduled to collect from the customer. |
| `picked_up_at` | timestamptz | When the driver collected the items. |
| `received_at` | timestamptz | When the warehouse physically received the returned goods. |
| `inspected_at` | timestamptz | When warehouse staff finished inspecting all returned items. |
| `put_away_at` | timestamptz | When the returned items were shelved, repaired, or disposed. |
| `refunded_at` | timestamptz | When the refund was processed. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |
| `created_by` / `updated_by` | varchar | Standard audit user fields. |

---

### `return_items` — individual items inside a return

One row per product being returned.

| Column | Type | Plain meaning |
|---|---|---|
| `return_item_id` | UUID PK | Unique ID. |
| `return_id` | UUID FK | The return request this item belongs to. |
| `order_line_id` | uuid | The original order line this item came from — used to calculate the refund amount. |
| `sku` | varchar | Product code. |
| `product_name` | nvarchar | Product display name. |
| `barcode` | varchar | Barcode for scanning at the warehouse dock. |
| `quantity` | decimal | How many units the customer is returning. |
| `unit_of_measure` | varchar | Counting unit. |
| `unit_price` | decimal | Price per unit from the original order — used to calculate the refund. |
| `currency` | varchar | Currency. |
| `item_reason` | varchar | Return reason specific to this item. |
| `condition` | varchar | Physical condition assessed at inspection: `Resellable` (back on shelf), `Repairable` (needs repair), `Dispose` (write off). |
| `put_away_status` | varchar | `Pending`, `PutAway`. |
| `assigned_sloc` | varchar | Storage location code where the item was placed after inspection — e.g. `B-05`, `REPAIR-BAY-2`, `DISPOSAL`. |
| `payment_method` | varchar | How the refund for this item will be returned to the customer. |
| `inspected_at` / `put_away_at` | timestamptz | Stage timestamps. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |

---

### `return_put_away_logs` — immutable put-away audit log

Records the physical act of shelving each returned item. Used for traceability if an item goes missing or a dispute arises.

| Column | Type | Plain meaning |
|---|---|---|
| `log_id` | UUID PK | Unique ID. |
| `return_id` | UUID FK | The return this entry belongs to. |
| `return_item_id` | UUID FK | The specific item put away. |
| `sku` | varchar | Copied here so the log is readable without JOINs. |
| `assigned_sloc` | varchar | Where the item was placed. |
| `condition` | varchar | Condition recorded at put-away time. |
| `quantity` | decimal | Units placed in this action. |
| `performed_by` | varchar | Warehouse staff member who placed the item. |
| `performed_at` | timestamptz | Exact time the put-away was confirmed. |

---

### `return_refunds` — refund payment record

The financial record of the money being returned to the customer.

| Column | Type | Plain meaning |
|---|---|---|
| `refund_id` | UUID PK | Unique ID. |
| `return_id` | UUID FK | The return this refund settles. |
| `refund_amount` | decimal | Total amount to be refunded. |
| `currency` | varchar | Currency. |
| `refund_method` | varchar | How the money is returned: `CreditCard` (reversed to card), `BankTransfer`, `StoreCredit` (added to loyalty wallet). |
| `status` | varchar | `Pending`, `Processed` (sent to gateway), `Failed`. |
| `reference_no` | varchar | Transaction reference from the payment gateway confirming the refund was sent. |
| `processed_at` | timestamptz | When the refund was submitted to the gateway. |
| `created_at` | timestamptz | When the refund record was created. |

## Module 4 — Configuration (schema: `config`)

---

### `store_locations` — every store and distribution centre

One row per physical location where OMS is active. Every order, PO, and transfer references a row here.

| Column | Type | Plain meaning |
|---|---|---|
| `store_id` | UUID PK | OMS's unique ID for this location. |
| `source_bu` / `source_loc` | varchar | Legacy system codes — used to map data during migration. |
| `store_name` | nvarchar | Display name — e.g. `Central DC`, `Store A — Sukhumvit`. |
| `business_unit` | varchar FK | Which brand group operates this store. References `business_units`. |
| `is_active` | bool | `true` if this store is currently accepting orders. |
| `is_rolled_out` | bool | `true` if OMS is live at this store. `false` means the store uses the old system. |
| `address1` → `longitude` | various | Physical address and GPS coordinates. Used for routing and customer-facing store locator. |
| `mobile_phone` | varchar | Store contact number for operations queries. |
| `email` | varchar | Store contact email. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |

---

### `business_units` — brands and business groups

The brands or subsidiary companies that operate stores (e.g. Tops Supermarket, Central Food Hall). A store belongs to exactly one business unit.

| Column | Type | Plain meaning |
|---|---|---|
| `bu_id` | UUID PK | Unique ID. |
| `bu_code` | varchar UK | Short code — e.g. `TOPS`, `CFH`. Used in reports and routing rules. |
| `bu_name` | nvarchar | Full brand name — e.g. `Tops Supermarket`. |
| `company_code` | varchar | Accounting company code used in the ERP system (e.g. SAP). |
| `company_name` | nvarchar | Legal registered company name for invoicing. |
| `is_active` | bool | Whether this BU is currently operating. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |

---

### `rollout_policies` — controls when OMS goes live at a store

Controls whether a store uses full OMS or falls back to the legacy system.

| Column | Type | Plain meaning |
|---|---|---|
| `policy_id` | UUID PK | Unique ID. |
| `store_id` | UUID FK | The store this policy applies to. |
| `is_rolled_out` | bool | Whether OMS is currently the active system at this store. |
| `integration_path` | varchar | Which mode is active: `Full` (full OMS), `LegacyFallback` (routing through old system). |
| `effective_from` | timestamptz | When this policy came into effect. |
| `effective_to` | timestamptz | When this policy expires. `null` means it is indefinite. |
| `updated_by` | varchar | Who last changed this policy. |
| `updated_at` | timestamptz | When it was last changed. |

---

### `fulfillment_routing_rules` — automatic routing decisions

When an order arrives, OMS uses these rules to decide how to route it — does it need a delivery slot? a TMS shipment?

| Column | Type | Plain meaning |
|---|---|---|
| `rule_id` | UUID PK | Unique ID. |
| `channel_type` | varchar | Which sales channel this rule applies to: `Web`, `App`, `POS`, `CallCenter`. |
| `fulfillment_type` | varchar | Which fulfillment type: `Delivery`, `Express`, `ClickAndCollect`. |
| `business_unit` | varchar | Which BU this rule applies to. `null` means the rule applies to all BUs. |
| `requires_booking` | bool | `true` if orders must have a delivery slot booked before progressing. |
| `requires_tms` | bool | `true` if orders need a TMS shipment (i.e. a driver must be dispatched). |
| `initial_pick_status` | varchar | What status the order moves to when picking begins. |
| `priority` | int | When multiple rules match, lower number wins. |
| `is_active` | bool | Whether this rule is currently applied. |
| `created_at` | timestamptz | When the rule was created. |

---

### `notification_templates` — message templates for customers and staff

### `purchase_orders` — buying goods from a supplier

A Purchase Order (PO) is raised when the business needs to restock from a supplier.

| Column | Type | Plain meaning |
|---|---|---|
| `purchase_order_id` | UUID PK | OMS's unique ID for this PO. |
| `po_number` | varchar UK | Human-readable PO number printed on delivery paperwork — e.g. `PO-001`. |
| `supplier_id` | uuid | Who is delivering the goods. References the external supplier registry (not stored in OMS). |
| `store_id` | UUID FK | Which store or DC is receiving this delivery. References `config.store_locations`. |
| `status` | varchar | `Created` → `PartiallyReceived` → `FullyReceived` → `Closed`. |
| `goods_receive_no` | varchar | Warehouse goods receipt number (GRN) issued when goods arrive at the dock. Set by WMS. |
| `created_at` / `updated_at` | timestamptz | Standard audit timestamps. |
| `updated_by` | varchar | Who last updated this PO. |

### `purchase_order_lines` — individual products on a purchase order

One row per product on the PO.

| Column | Type | Plain meaning |
|---|---|---|

| `received_at` | timestamptz | When these goods were received at the warehouse dock. |
| `put_away_at` | timestamptz | When goods were placed on the shelf and made available for picking. |
